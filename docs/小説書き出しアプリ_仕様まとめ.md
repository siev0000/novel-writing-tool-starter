# 小説書き出し用アプリ 仕様まとめ

## 1. アプリ概要

このアプリは、小説の「書き出し」を作るための補助アプリである。

設定メモ、登場人物、タグ、相関図、用語まとめ、プロット、本文を管理し、それらをもとに書き出し候補を作成・保存・比較できるようにする。

最終的には、書き出し候補を本文に反映し、本文編集・行間メモ・投稿前プレビュー・投稿用エクスポートまで行える小説制作補助ツールとする。

---

```txt
小説書き出し用アプリ
├─ 作品管理
├─ 設定用メモ
│  ├─ 登場人物
│  ├─ タグ
│  ├─ 相関図
│  └─ 用語まとめ
│
├─ プロット
│  ├─ 作品プロット
│  └─ 話プロット
│     └─ シーン
│
├─ 書き出し作成
│  ├─ 作品プロットから作る
│  ├─ 話プロットから作る
│  ├─ シーンから作る
│  ├─ 登場人物から作る
│  ├─ タグから作る
│  └─ 用語から作る
│
└─ 本文編集
   ├─ 本文エディタ
   ├─ 行間メモ
   ├─ 投稿前プレビュー
   ├─ 投稿用エクスポート
   ├─ 本文内検索
   └─ 編集履歴
```

## 3. 基本方針

```txt
設定メモ = 素材
プロット = 流れ
書き出し = 本文化の入口
本文編集 = 実際に書く場所
```

プロットは、作品全体と話ごとで分ける。

```txt
作品プロット = 作品全体の地図
話プロット = 各話の詳細ルート
シーン = 本文に変換する具体的な場面
```

### 実装ルール

この章の内容は、画面追加・画面修正・モーダル追加・入力項目追加を行う際の必須ルールとする。

#### UI共通ルール

```txt
- 入力項目の表示は原則すべて統一する
- 標準の入力表示は「メモ」項目と同じレイアウトを基準にする
- 表示形式は「ラベル + 余白 + 下線」とし、通常の枠線付き入力欄を基本にしない
- 個別機能だけ別レイアウトを作らず、追加項目や概要項目も同じ入力ルールに合わせる
- 編集ボタンは右上配置の「✎」で統一する
- 編集中のみ追加・削除などの破壊的操作を表示する
- 削除操作は必ず確認を挟む
- 削除確認では「何を削除するか」と「削除後の影響」が分かる文言を出す
- 左上の戻るは画面遷移専用とし、編集の巻き戻しには使わない
- 編集の巻き戻し機能を入れる場合は「元に戻す / やり直す」をセットで用意する
- 画面遷移の戻ると編集履歴の巻き戻しは、名称も配置も分けて混同させない
```

## 6. 登場人物

### 目的

キャラクターのプロフィールを管理する。

### 方針

キャラクターのプロフィール項目は、初期状態でデフォルト項目を用意する。使用者は後からプロフィール項目を自由に追加できる。

### デフォルトプロフィール項目

```txt
登場人物
- 名前
- 読み
- 種族
- 所属
- 立場
- 物語上の役割
- 性格
- 目的
- 弱点
- 秘密
- 口調
- 特技
- 外見
- 初登場シーン
- 関連タグ
- メモ
```

### 追加可能なプロフィール項目例

```txt
- 好物
- 嫌いなもの
- 出身地
- 家族構成
- 二つ名
- 戦闘スタイル
- 信仰
- 所持品
- トラウマ
- 口癖
```

### 特技

キャラクターが得意なことを記録する。

```txt
例：
剣術、魔法、交渉、隠密、料理、鑑定、機械修理、嘘を見抜く
```

### 外見

キャラクターの見た目や印象を記録する。

Ver.1では大きなテキスト欄として扱う。

```txt
例：
銀髪で青い目をした少年。細身だが目つきは鋭い。普段は古びた黒い外套を着ている。
```

Ver.2以降で細分化する。

```txt
外見詳細
- 髪型
- 髪色
- 目の色
- 身長
- 体格
- 服装
- 装備
- 傷跡
- 種族的特徴
- 第一印象
```

## 7. 登場人物とプロットの紐づけ

登場人物は、作品全体に登録するだけでなく、作品プロット・話プロット・シーンごとにも設定できるようにする。

```txt
登場人物の紐づけ
├─ 作品に登場する人物
├─ 作品プロットで重要な人物
├─ 話プロットに登場する人物
└─ シーンに登場する人物
```

### 役割の違い

```txt
作品の登場人物
= その作品に存在するキャラクター全体

作品プロットの登場人物
= 物語全体に関わる主要人物

話プロットの登場人物
= その話に登場する人物

シーンの登場人物
= その場面に実際に出る人物
```

### 例

```txt
作品の登場人物
- アルト
- リリア
- 神官
- 村長
- 黒幕

作品プロットの重要人物
- アルト
- リリア
- 黒幕

第1話の登場人物
- アルト
- リリア
- 神官
- 村人

シーン1の登場人物
- アルト
- 神官
- 村人
```

## 8. キャラクター詳細の登場シナリオ一覧

キャラクター詳細画面には、そのキャラクターが登場する話・プロット・シーンを一覧表示する。

```txt

├─ プロフィール
├─ タグ
├─ 関係性
├─ 登場シナリオ
└─ メモ
```

### 登場シナリオ一覧に表示する項目

```txt
登場シナリオ
- 第何話
- 話タイトル
- プロット名
- シーン名
- 登場区分
- メモ
```

### 例

```txt

第1話：女神を知らない少年
プロット名：祈りの儀式
シーン：村の広場での儀式
登場区分：主要登場
```

```txt

登場シナリオ一覧
↓
第1話：女神を知らない少年 を押す
↓
第1話プロット詳細へ移動
```

または、シーン名を押してシーン詳細へ移動する。

## 9. 登場区分

登場人物がその話・シーンでどの程度関わるかを設定できるようにする。

```txt
登場区分
- 主要登場
- サブ登場
- 名前のみ
- 回想
- 会話内のみ
- 未登場だが関係あり
```

### 登場区分の用途

```txt
主要登場
= その話やシーンの中心人物

サブ登場
= その場に登場するが中心ではない人物

名前のみ
= 本人は出ないが名前だけ出る

回想
= 過去の記憶や回想で出る

会話内のみ
= 会話の中で触れられるだけ

未登場だが関係あり
= 直接出ないが、事件や伏線に関係している
```

## 11. 相関図

### 目的

登場人物同士の関係を管理する。

最初はビジュアル表示ではなく、関係リスト形式で管理する。

### 基本項目

```txt
相関図
- 人物A
- 人物B
- 関係性
- AからBへの感情
- BからAへの感情
- 表向きの関係
- 裏の関係
- 関係の変化予定
- 関連タグ
- メモ
```

### 例

```txt
人物A：アルト
人物B：聖女リリア

関係性：監視対象
AからBへの感情：警戒
BからAへの感情：興味、疑念

表向きの関係：
神殿に呼ばれた少年と聖女

裏の関係：
リリアはアルトが女神の加護を持たないことに気づいている

関係の変化予定：
警戒 → 協力 → 信頼
```

```txt

↓

↓

↓

↓

```

仕様。

```txt
書き出し候補には、選択した用語を自然に含められる。
ただし、ネタバレ情報は指定しない限り書き出しには含めない。
```

## 18. 書き出し作成

### 書き出し作成元

## 31. Google Drive バックアップ方針

Google Drive 連携は、ユーザー本人の Google アカウントでログインして利用する。

作品データは、ユーザー本人の Google ドライブ内に保存する。

保存先は原則として、Google Drive API のアプリ専用保存領域を使用する。

アプリは、ユーザーの Google ドライブ全体を参照・編集するのではなく、このアプリが作成したバックアップデータのみを保存・読み込みする。

バックアップは JSON 形式で保存し、別端末でも同じ Google アカウントでログインすれば読み込めるようにする。

### Ver.1

```txt
- 自分のGoogleアカウントだけでテスト
- 手動バックアップ
- 手動読み込み
- JSON 1ファイル保存
```

## 32. Markdown 出力の将来対応メモ

将来的に情報量が増えた場合は、以下の分割出力も検討する。

```txt
- 人物ごとに個別 Markdown 出力
- 用語ごとに個別 Markdown 出力
- 章ごと / 話ごとに個別 Markdown 出力
```
