# AI渡し用 JSON生成メモ

このメモは、通常メモから `novel-writing-tool-project` 形式の JSON を生成する時だけ使う。

このファイルには、アプリ実装方針や開発ルールは入れない。

---

## 1. 目的

通常メモを、作品読込用の JSON に変換する。

---

## 2. 出力ルール

```txt
- 出力は JSON のみ
- 説明文、前置き、コードフェンスは付けない
- 書かれていない設定は補完しない
- 曖昧な情報は memo に入れる
- 不明項目は空文字、空配列のままにする
- 同一と断定できない人物・タグ・用語は統合しない
```

---

## 3. 使うテンプレート

テンプレートは以下を使う。

```txt
docs/AI用_最小JSONテンプレート.json
```

この構造を維持したまま中身を埋める。

---

## 4. 読み取りルール

### 人物

```txt
- 人名があれば characters に入れる
- 読みがあれば ruby に入れる
- 性格、目的、所属などは対応する項目へ入れる
- 行き先が曖昧な説明は memo に入れる
```

### タグ / 用語

```txt
- 分類・検索用の短い概念は tags に入れる
- 長い説明を持つものも tags に入れる
- 用語扱いのものは type を「用語」にする
- 概要、特徴、短い説明、公開情報は summaryItems に分けてよい
```

### プロット

```txt
- 作品全体は workPlot
- 章単位は chapters
- 話単位は episodes
- シーン単位は scenes
- 迷う内容は上位の memo に入れる
```

---

## 5. summaryItems ルール

```txt
- 「概要:」→ title: "概要"
- 「特徴:」→ title: "特徴"
- 「短い説明:」→ title: "短い説明"
- 「公開情報:」→ title: "公開情報"
- 見出しが無い説明文 → title: "概要"
```

---

## 6. 生成してはいけないこと

```txt
- AI が勝手に人物関係を追加する
- AI が未記載のタグを追加する
- AI が設定の穴を創作で埋める
- AI が既存データとの対応を推測だけで決める
```

---

## 7. AIへの指示文

```txt
以下のメモを、小説書き出しアプリ用の JSON に変換してください。

条件:
- 出力は JSON のみ
- docs/AI用_最小JSONテンプレート.json の構造を維持する
- 書かれていない設定は補完しない
- 曖昧な内容は memo に入れる
- 人物、タグ、プロットを分けて整理する
- 用語扱いの項目は tags の type を「用語」にする
- 概要、特徴、短い説明、公開情報は必要なら summaryItems に分ける
- 同一と断定できないものは別データとして扱う
```
